# 📚 전자책 프로젝트 통합 관리 저장소 (A-E-Book Project Repository)

이 저장소는 다수의 전자책 기획, 원고 집필, PDF 제작, 판매 및 마케팅 자료를 체계적으로 관리하고 무한히 확장할 수 있도록 설계된 표준 폴더 구조를 사용합니다. 
초보 작성자나 파트너와 협업하는 작업자 모두가 한눈에 전체 흐름을 이해하고 바로 작업에 착수할 수 있습니다.

- **용도**: 여러 전자책을 만들 때 중복해서 쓰이는 공통 디자인 템플릿, 표지 가이드, PDF 폰트/레이아웃 세팅, 고객 응대(CS) 템플릿 등을 관리합니다.
- **주요 포함 파일**:

  - [12_PDF디자인가이드.md](file:///C:/안티그래비티/전자책/00_공통자료/12_PDF디자인가이드.md) (PDF 본문 조판 및 레이아웃 가이드)

### 2. `01_유튜브음악채널수익화가이드2026` (개별 전자책 프로젝트 폴더)
각 책의 폴더 내부에는 제작 흐름에 맞춘 8개의 하위 표준 폴더가 정의되어 있습니다:
- **`01_기획`**: 기획안, 목차 구성안, 레퍼런스 링크, 타겟 분석 문서 보관
- **`02_원고`**: 실제 본문 마크다운(.md) 또는 텍스트 문서 집필 공간 (챕터별 구분)
- **`03_시각자료`**: 본문에 들어가는 스크린샷, 도표, 인포그래픽, 캡처 이미지
- **`04_표지`**: 책 표지 PSD/PNG 파일, 크몽 썸네일(652x488) 등 비주얼 에셋
- **`05_PDF제작`**: HTML/CSS 기반 PDF 변환 소스 코드, 빌드 스크립트, 중간/테스트 PDF 파일
- **`06_크몽상세페이지`**: 크몽 상품 페이지 기획안, 상세 이미지 원본, 업로드 미리보기 자료
- **`07_홍보콘텐츠`**: 해당 책 전용 블로그 홍보글, 유튜브 숏츠 대본 등 홍보 원고
- **`08_수정이력`**: 버전업 시 변경 내역 기록(Changelog) 보관

### 3. `02_얼굴없이시작하는블로그수익화`
- **용도**: 2호 책의 신규 집필을 위해 1호 책과 동일한 8개 표준 폴더 구조를 미리 구성해 놓은 빈 프로젝트 폴더입니다.

## 🚀 초보자를 위한 3호 책 추가 생성 자동화 가이드

앞으로 3호, 4호 전자책을 추가로 기획하실 때, 표준 폴더 구조를 수동으로 하나씩 만들기 번거로우실 것입니다. 이를 위해 자동 생성할 수 있는 PowerShell 스크립트를 제공합니다.

### 실행 방법:
1. PowerShell 콘솔을 엽니다.
2. 프로젝트 루트 폴더(`C:\안티그래비티\전자책`)로 이동합니다.
3. 다음 명령어를 실행하여 템플릿 구조를 자동 생성합니다:
   ```powershell
   $bookName = "03_원하는책프로젝트이름"
   $subDirs = @("01_기획", "02_원고", "03_시각자료", "04_표지", "05_PDF제작", "06_크몽상세페이지", "07_홍보콘텐츠", "08_수정이력")
   foreach ($sub in $subDirs) { New-Item -ItemType Directory -Force -Path "$bookName/$sub" }
   ```
